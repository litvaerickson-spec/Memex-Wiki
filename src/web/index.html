Поиск по памяти

Точечный семантический поиск по концептам методом RLM-фильтрации

💡 Справка
Здесь вы можете задать любой вопрос по вашим проектам. Система RLM автоматически отсечет лишние чанки документов, предоставив только краткую суть с точной ссылкой на источник (source: файл).

Как работает RLM-поиск?

Система нарезает файлы на чанки по 3000 токенов и опрашивает выбранного провайдера (Gemini или Ollama) в строго однопоточном режиме. Это исключает зависание компьютера и перегрузку контекста.

О системе Memex-Wiki

Архитектура, принципы работы и преимущества гибридной памяти

Назначение гибридной памяти

Система спроектирована как локальный цифровой «второй мозг» (Second Brain). Она решает фундаментальные проблемы классического динамического RAG (Retrieval-Augmented Generation): зависимость от интернет-подключения, высокая стоимость облачных запросов, утечка конфиденциальных данных и «забывание информации в середине контекста» при передаче больших объемов кода.

Как это работает простыми словами

🔄 Процесс в 4 шага
1

Загрузка файлов:

Вы добавляете рабочие файлы проектов в папку.

↓
2

Атомизация знаний:

ИИ читает файлы один раз и выделяет из них ключевые понятия.

↓
3

Связывание ссылками:

Каждое понятие ИИ связывает с другими понятиями.

↓
4

Семантический поиск:

При запросе ИИ находит нужные понятия на карте и собирает из них ответ.

🕸️ Роль графа связей

Представьте карту дорог, где города — это темы или файлы ваших проектов (узлы графа), а дороги — это связи между ними. Вместо слепого поиска по тексту, ИИ мгновенно перемещается по этим дорогам от темы к теме. Это позволяет ему удерживать картину вашего проекта целиком, не галлюцинировать и находить неочевидные взаимосвязи.

Заметки
1. Накопление (по Карпатому)

Инкрементальная компиляция знаний: данные перерабатываются ИИ в момент добавления (ingest). Файлы сжимаются, выделяются ключевые концепты и встраиваются в связанные Markdown-страницы Obsidian.

2. Вычисление (по MIT RLM)

Контекст как среда выполнения (Sandbox): вместо передачи сырого текста документов в промпт, Root-модель запускает локальный Python-код для поиска, фильтрации и точечного опроса чанков.

Ключевые принципы и компоненты

  • 📂
    Хранилище Obsidian (MD + YAML): Использование Markdown-файлов для заметок. Связи [[Concept]] выступают ребрами графа знаний, а YAML-секция в заголовке хранит метаданные для быстрого разбора кодом.
  • 🐙
    Контроль версий памяти (Git): Все изменения памяти логируются в репозитории. Вы можете отследить, какие знания ИИ перезаписал, и сделать откат в случае ошибок.
  • 🤖
    Свобода выбора (Gemini / Ollama): Возможность работать полностью локально без интернета на моделях Qwen/Llama с помощью Ollama, либо переключаться на сверхдешевый Gemini API для объемных задач.
  • 🛡️
    Безопасность и Редактирование секретов: Бэкенд автоматически вырезает пароли, API-ключи и строки подключения баз данных при обработке сырых материалов, защищая ваши данные.

Основные преимущества

🔒 100% Локально и Приватно При выборе Ollama все данные остаются на вашем компьютере. Информация о ваших проектах никогда не утечет наружу.
🧠 Устойчивость к галлюцинациям За счет пошаговой компиляции знаний и точного поиска по регулярным выражениям модель синтезирует ответы только на базе реальных фактов.
⚡ Экономия контекста ИИ Рекурсивная декомпозиция MIT RLM передает ИИ только отфильтрованные строки, решая проблему лимитов токенов и зависания.
🔗 Синергия базы знаний Автоматическая генерация связей [[Concept]] строит визуальную граф-сеть ваших проектов для удобного ручного или автоматического анализа.
