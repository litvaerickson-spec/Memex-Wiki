## [Unreleased] — v0.2 (в разработке)

### Исправлено
- **Взаимная блокировка (Deadlock) в тихой индексации** — исправлена критическая ошибка в `src/web_server.py`, приводившая к зависанию сервера на первом же файле. Блокировки переведены на `threading.RLock()`, а логирование `_log()` вынесено из критических секций.
- **Мерцание экрана при переключении вкладок** — полностью решено за счет приостановки цикла анимации `tick()` и вызова `dispose()` для WebGL-рендерера Three.js при деактивации вкладки с 3D-графом.
- **Высокая нагрузка и перегрев процессора** —
  - Рендеринг 3D-графа теперь полностью отключается, когда пользователь не находится на вкладке графа.
  - Добавлена интеллектуальная обрезка текста импортируемых файлов до 15 000 символов (в начале и конце файла), что защищает Ollama от бесконечных вычислений на гигантских файлах.
- **Оптимизация задержки тихой индексации** — дефолтная пауза между файлами во время тихой индексации уменьшена с 30 до 10 секунд (ETA сокращено в 3 раза).

### Добавлено
- **Живой прогресс сканирования (SSE)** — новый эндпоинт `/api/scan_raw_stream`
  использует Server-Sent Events: браузер получает события в реальном времени
  по мере обхода папок (`status`, `progress`, `found_new`, `found_modified`, `done`).
- **Панель прогресса сканирования** — вместо статичного спиннера появилась
  живая панель с пульсирующей полосой прогресса, текущей папкой в monospace,
  и тремя счётчиками (Папок / Новых / Изменённых), обновляющимися на лету.
- **Файлы появляются по мере нахождения** — каждый найденный файл добавляется
  в список сразу с анимацией fade-in, не дожидаясь окончания всего сканирования.
- **Итоговая карточка сканирования** — после завершения появляется
  постоянная карточка: ✓ заголовок, время завершения, 4 цифры
  (Папок / Новых / Изменённых / Актуальных), цветная рамка.
- **Тепловой мониторинг v2 (многоисточниковый)** — полностью переписан
  `/api/thermal`, теперь комбинирует три независимых источника данных:
  1. **CPU idle%** — `top -l 2 -s 1` (два замера, берётся второй — реальный
     дельта за 1 сек, а не накопленное с загрузки ОС как в `-l 1`).
  2. **Температура батареи** — `ioreg -c AppleSmartBattery` без sudo,
     читает `VirtualTemperature` (точнее при нагрузке) и `Temperature`
     (0.01°C → °C). MacBook: <35°C норма, 35–42° тепло, 42–50° горячо, >50° критично.
  3. **Sustained load** — 5-минутный load average / кол-во ядер: учитывает
     длительную нагрузку, которая не видна в мгновенном CPU idle%.
  - Итоговый уровень = максимум из трёх источников.
  - UI-бейдж показывает `CPU idle: 62% · 🌡 38°C`, tooltip — детали всех источников.
  - Порог «холодно» повышен с 55% до **70% CPU idle**.
  - Фронтенд ждёт реального остывания (re-poll каждые 5 сек), а не фиксированные N секунд.
  - Новые пороги CPU idle: >70% cool · 55–70% warm · 35–55% hot · <35% critical.
- **3D граф — оптимизация производительности** при 1000+ узлах:
  - Убран обход всех узлов в каждом кадре (`rotation.y/x +=`, hover scale/emissive)
    — на 1000 нодах это тысячи матричных операций/кадр → лаги и пульсации.
  - Частицы на рёбрах: 0 при >200 узлах, 1 при малом графе.
  - `linkResolution` 6→4, `linkWidth` 0.8→0.6.
  - Hover-эффекты (`onNodeHover`) убраны — они вызывали пересчёт scale для всего дерева.
- **3D граф — камера**: начальная позиция вычисляется из числа узлов
  (`min(4000, max(400, N×15))`), `onEngineStop → zoomToFit(600, 60)` подгоняет
  камеру под реальный bbox. Орбита запоминает расстояние после fit (+8% запаса).
- **🌙 Тихая индексация** — новый режим пакетного импорта для работы в фоне
  без перегрева устройства:
  - Кнопка **🌙 Тихая индексация** появляется рядом с «Импортировать все» после скана.
  - Фоновый поток (`daemon=True`) с `os.nice(19)` — минимальный приоритет CPU,
    ОС отдаёт ресурсы всем другим процессам первым.
  - **30 секунд паузы** между каждым файлом (разбиты на 1-сек шаги, чтобы Стоп
    реагировал мгновенно).
  - Перед каждым файлом проверяет CPU idle: если < 70% — ждёт дополнительно
    (re-poll каждые 10 сек, максимум 120 сек).
  - Три новых API-эндпоинта:
    - `POST /api/ingest_quiet_start` — принимает `files[]` + `delay_sec`.
    - `GET  /api/ingest_quiet_status` — возвращает running/done/errors/current/log/eta.
    - `POST /api/ingest_quiet_stop` — останавливает поток.
  - Панель прогресса с фиолетовой полосой, счётчиками, ETA в минутах,
    живым логом (последние 20 строк), автопрокруткой.
  - При завершении кнопка «Стоп» превращается в «Закрыть».
  - Поллинг статуса каждые 10 сек — браузер не держит открытых соединений.
  - При перезагрузке страницы панель восстанавливается если процесс ещё идёт.

- **Панель пакетного импорта с управлением** — живая панель во время импорта:
  - Прогресс-бар с процентом выполнения
  - Имя текущего файла (monospace)
  - Тепловой бейдж в реальном времени (🟢/🟡/🟠/🔴 + CPU idle%)
  - Счётчики: Готово / Ошибок / Осталось
  - Кнопки **⏸ Пауза** и **✕ Стоп** — можно приостановить или прервать
    в любой момент без потери уже обработанных файлов
  - Оранжевый баннер при активной паузе
- **Итоговая карточка импорта** — после завершения или остановки показывает
  время выполнения, Файлов / Успешно / Ошибок.
- **CSS-анимация `scanPulse`** — индетерминированная прогресс-полоса
  анимируется слева направо как шиммер.
- **Новые ключи локализации (RU/EN)**:
  `scan_progress_label`, `scan_dirs_label`, `scan_new_label`,
  `scan_modified_label`, `scan_uptodate_label`

### Исправлено
- **Кнопки «Стоп» и «Пауза» не реагировали** во время импорта большого файла —
  браузер ждал ответа от сервера (`await fetch`) и не мог перерисовать UI.
  Исправление:
  - Добавлен `AbortController` — нажатие «✕ Стоп» немедленно обрывает текущий
    HTTP-запрос к `/api/ingest`, не дожидаясь ответа LLM.
  - Кнопки получают элементы через `id` (`ingest-pause-btn`, `ingest-stop-btn`)
    и обновляются точечно, без перерисовки всей панели через `innerHTML`.
  - «✕ Стоп»: мгновенно показывает `🛑 Останавливаем...`, блокирует обе кнопки.
  - «⏸ Пауза» во время запроса: показывает `⏸ Пауза после текущего файла...`
    (запрос дожидается ответа, пауза активируется после него).
  - «⏸ Пауза» между файлами: кнопка сразу меняется на `▶ Продолжить`.
  - Статус-бейдж (`ingest-status-badge`) показывает
    `⏳ Отправляю в LLM... filename` пока идёт запрос.
  - `AbortError` не засчитывается как ошибка в счётчике.


### Изменено
- Функция `scanRawFiles()` переписана с `async/fetch` на `EventSource` (SSE).
- Эндпоинт `/api/scan_raw` оставлен без изменений для обратной совместимости.

### Безопасность
- Удалены все хардкодированные личные пути из исходного кода:
  - `src/memex_tools.py` — дефолтные значения `obsidian_vault`, `raw_dir`
  - `src/graph_linter.py` — дефолтное значение `obsidian_vault`
  - `src/web/index.html` — placeholder'ы и текстовые подсказки (RU + EN)
  - `config.example.yaml` — пример папки заменён на generic `MyObsidian`
- История git полностью переписана (`git filter-repo` + orphan squash):
  - Все 7 коммитов объединены в один «Initial release v0.1»
  - Автор коммитов анонимизирован: `Memex-Wiki Author <memex-wiki@users.noreply.github.com>`
  - Личные пути, имена папок и hostname удалены из истории
  - Force-push выполнен, старые объекты собраны сборщиком мусора

## [0.1.0] — 2026-06-25

### Первый публичный релиз

#### Добавлено (фронтенд — `src/web/index.html`)
- **Favicon** — светлый квадрат с буквой «W» в формате SVG data URI.
- **Переключатель тема** — кнопка Dark/Light с сохранением в `localStorage`.
- **Двуязычный интерфейс (RU/EN)** — полный словарь `translations`,
  переключатель языка, функция `applyTranslations()`.
- **Вкладка «Импорт»** — сканирование папок, список новых/изменённых файлов,
  поштучный и пакетный импорт, лог последних операций.
- **Вкладка «Линтер»** — цветной вывод ошибок/предупреждений с развёрнутыми
  рекомендациями что делать с каждым типом проблемы.
- **Вкладка «Настройки»** — поля для Gemini API ключа, выбор модели Ollama
  (динамический `<select>`), скачивание новой модели с прогресс-баром,
  пути к Obsidian Vault и Raw Folder, паттерны исключений,
  переключатели темы и языка.
- **Вкладка «О системе»** — описание архитектуры для не-технарей,
  интерактивная CSS-схема «Как это работает» (5 шагов),
  визуализация графа знаний с узлами и связями,
  объяснение концепции гибридной памяти.
- **Версия и ссылка на GitHub** — левый нижний угол сайдбара, v0.1.
- **Исправление контрастности светлой темы** — тексты узлов графа,
  тултипы, кнопки, фоны карточек.

#### Добавлено (бэкенд — `src/web_server.py`)
- Эндпоинт `GET /api/config` — чтение конфига (без секретных ключей в ответе).
- Эндпоинт `POST /api/config` — запись изменений в `config.yaml`.
- Эндпоинт `GET /api/scan_raw` — сканирование raw folder, возврат JSON.
- Эндпоинт `POST /api/ollama_pull` — фоновая загрузка модели Ollama.
- Эндпоинт `GET /api/ollama_pull_status` — статус и % прогресс загрузки.

#### Добавлено (инфраструктура)
- `config.example.yaml` — шаблон конфигурации без личных данных.
- `.gitignore` — исключает `config.yaml`, `.venv`, `__pycache__`, логи, архивы, ТЗ.
- `setup_env.sh` — скрипт установки окружения для macOS/Linux.
- `src/launch.sh` — скрипт запуска с динамическим определением пути (без абсолютных путей).
- `README.md` — двуязычная документация (RU + EN):
  - Описание проекта и ключевые возможности
  - Схема архитектуры (ASCII)
  - Пошаговые инструкции установки для macOS, Windows, Linux
  - Скриншоты тёмной и светлой тем
  - Раздел безопасности
- `LICENSE` — MIT License.
- `Dark_theme.png`, `White_theme.png` — скриншоты интерфейса.

#### Безопасность
- Все личные абсолютные пути и токены заменены шаблонами в публичных файлах.
- `config.yaml` добавлен в `.gitignore` и не попадает в репозиторий.
- Архивные папки и локальные ТЗ-документы удалены из git-истории.
